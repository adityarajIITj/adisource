2. Knowledge-Based Agents

What is a Logical Agent?

A logical agent is an intelligent system that:

  • Perceives the real environment
  • Creates a representation of that world
  • Performs reasoning and calculations on this representation
  • Makes decisions and takes actions based on logical inference
[Insert diagram: Logical Agent Architecture - Real Environment → Representation → Knowledge Base → Reasoning → Action]

The Knowledge Base (KB)

At the heart of every knowledge-based agent is the Knowledge Base (KB):

Component Description Example
Knowledge Base (KB) A collection of sentences representing prior knowledge Facts about mathematics, physics, or domain-specific information
Sentences Individual statements or facts in the KB "All humans are mortal", "$x + y = 4$"
Language The formal system used to express sentences Natural language, mathematical notation, programming languages
Professor mentioned in class: "In a knowledge-based agent, what is this representation? This representation is done by some knowledge - a knowledge base. This is the prior knowledge that the agent has. For example, it can be knowledge from this AI course, or any knowledge base, and generally we denote it by KB."

How Does a Knowledge-Based Agent Work?

🔄 Agent Operation Cycle
  1. Represent the Real World: The complex real world is abstracted into symbols and sentences
  2. Store Prior Knowledge: The agent has a knowledge base with existing facts
  3. Receive Questions: When asked a question, the agent queries its KB
  4. Reason and Infer: The agent uses logical reasoning to derive new knowledge
  5. Provide Answers: Conclusions are reached through step-by-step logical steps

The Tell-Ask Interface

Knowledge-based agents interact through two primary operations:

  • TELL: Adding new sentences/knowledge to the KB
  • ASK: Querying the KB for information or answers
Example: Mathematical Reasoning

Question: Solve $x + y = 4$ where $x, y$ are natural numbers.

Knowledge Base: Contains knowledge about addition of natural numbers.

ASK Operation: Query the KB for solutions.

Inference Engine: Reasons through possibilities.

Answer: $(x=1, y=3)$, $(x=2, y=2)$, $(x=3, y=1)$

The reasoning capability is one of the most important aspects of an intelligent agent. This is similar to theorem proving - you follow step-by-step logical steps."

Reasoning Capability

The reasoning capability is what distinguishes intelligent agents:

  • Using prior knowledge to derive new knowledge
  • Following logical steps systematically
  • Similar to mathematical theorem proving
  • Making conclusions even when answers aren't explicitly stored

3. Logic Fundamentals

What are Sentences?

A sentence in the context of logical agents is more general than everyday language sentences:

🔤 Types of Sentences
  • Natural Language: "I eat rice"
  • Mathematical: "$x + y = 4$"
  • Programming: if (x > 0) then print("positive")

Sentences are expressed through languages, which can be:

  • Natural languages (English, Hindi, etc.)
  • Programming languages (C++, Python, Java, etc.)
  • Mathematical languages (algebra, calculus, logic)
  • And many more formal systems
Professor mentioned in class: "Don't just take it as your English language sentence or any other language sentences. Sentences are more general. They are expressed through languages - it can be English, our mother tongue, programming languages like C++, Python, mathematical expressions, and many more. Everything is a language, and wherever you have a language, you can create sentences."

3.1 Syntax and Semantics

Every formal language has two critical components:

Aspect Definition Focus Example
Syntax Rules for forming valid sentences Structure and form "I eat rice" ✓ vs "Rice eat I" ✗
Semantics The meaning of sentences Interpretation and truth Whether "I eat rice" is true or false in a given situation

Syntax: The Rules of Formation

Syntax defines the positions or structure of elements in a sentence:

Natural Language Example

✅ Correct: "I eat rice" - Subject → Verb → Object

❌ Incorrect: "Rice eat I" - Wrong word order

Mathematical Language Example

✅ Correct: $x + y = 4$

❌ Incorrect: $x \, y + 4$ - Malformed expression

Professor mentioned in class: "For natural languages, it's the position of the words. We don't write 'rice eat I' because that will be syntax-wise incorrect. Similarly, in mathematical language, we write $x + y = 4$, not $x \, y + 4$. These positions matter!"

Semantics: The Meaning

Semantics concerns what a sentence means - whether it's true or false in a particular context:

  • Every sentence must have meaning
  • Meaning determines truth or falsity
  • Context (the "world" or "model") matters
Semantic Analysis Example

Sentence: $x + y = 4$ (where $x, y \in \mathbb{N}$)

Question: In which "worlds" or environments is this sentence true?

Answer: This sentence has meaning - it's asking about natural number pairs that sum to 4.

Professor mentioned in class: "How are you deciding these positions are correct? That depends on the meaning of the sentence. 'I eat rice' is meaningful to us. $x + y = 4$ means something to us in mathematical concept, but $x \, y + 4$ doesn't have meaning. Semantically, every sentence has to mean something."

3.2 Models and Truth Values

What is a Model?

A model is a "world" or "environment" where sentences can be evaluated as true or false.

Understanding Models with Examples

Sentence α: $x + y = 4$ (where $x, y \in \mathbb{N}$)

Model m₁: $x = 2, y = 2$

→ Does m₁ satisfy α? $2 + 2 = 4$ ✅ YES! So m₁ is a model of α.

Model m₂: $x = 1, y = 3$

→ Does m₂ satisfy α? $1 + 3 = 4$ ✅ YES! So m₂ is a model of α.

Model m₃: $x = 3, y = 1$

→ Does m₃ satisfy α? $3 + 1 = 4$ ✅ YES! So m₃ is a model of α.

Non-model m₄: $x = 1, y = 1$

→ Does m₄ satisfy α? $1 + 1 = 2 \neq 4$ ❌ NO! So m₄ is NOT a model of α.

Set of All Models

We denote M(α) as the set of all models of sentence α:

$$M(\alpha) = \{m_1, m_2, m_3, \ldots\}$$

For $\alpha: x + y = 4$ where $x, y \in \mathbb{N}$:

$$M(\alpha) = \{(1,3), (2,2), (3,1)\}$$

Key Definition: A model is any "world" or assignment of values where the sentence α is true.

Professor mentioned in class: "All those worlds where the sentence α is true are called the models of α. Let's denote it by M(α) - the set of all or collection of all models of α. What is a model? A model is all the worlds where this sentence α is true, or this model will satisfy sentence α."

M(α) se hum denote karte hain α ke sabhi models ka set.

4. Inference and Reasoning

Now that we understand sentences, models, and meaning, let's explore how we create new sentences from existing knowledge through inference.

4.1 Entailment

Entailment is the fundamental concept that allows us to derive new knowledge from existing knowledge.

Formal Definition

We say that sentence α entails sentence β (written as α ⊨ β) if and only if:

$$\alpha \models \beta \iff M(\alpha) \subseteq M(\beta)$$

In words: Every world where α is true, β is also true.

Understanding Through Example

Entailment Example

Sentence α: "It is raining and it is cold"

Sentence β: "It is raining"

Question: Does α entail β?

Analysis:

  • In every world where "it is raining AND it is cold" is true, "it is raining" must also be true
  • Therefore, M(α) ⊆ M(β)
  • Conclusion: Yes! α ⊨ β
Mathematical Entailment Example

Sentence α: $x = 2$ and $y = 2$

Sentence β: $x + y = 4$

Analysis:

  • M(α) = {(2, 2)} - only one world where both conditions are true
  • M(β) = {(1,3), (2,2), (3,1)} - multiple worlds where sum equals 4
  • Is M(α) ⊆ M(β)? Yes! (2,2) ∈ M(β)
  • Conclusion: α ⊨ β
Professor mentioned in class: "Entailment is very necessary for inference. It means in every world where α is true, β is also true. This is represented as M(α) should be contained in M(β). This concept is crucial for creating new sentences from old knowledge."

Inference Process

The inference process is how we actually create new sentences:

[Insert diagram: KB (Knowledge Base) → Inference Engine (i) → New Sentence (α)]

We denote this as: KB ⊢i α

Reading: "From KB, using inference mechanism i, we derive sentence α"

Inference in Action

Knowledge Base: Knowledge about addition on natural numbers

Question: Solve $x + y = 4$

Inference Engine: Our brain/reasoning mechanism

Derived Sentence: $x = 2, y = 2$ (one solution)

Professor mentioned in class: "From using this knowledge base, we are creating new sentences. From old knowledge, you can think about it as creating new sentences. This is what the reasoning part or inference part is all about - through logical reasoning, creating another sentence."

4.2 Soundness

An inference mechanism must be reliable - this is where soundness comes in.

Formal Definition

An inference mechanism i is sound if:

If KB ⊢i α, then KB ⊨ α

In words: Every sentence that i produces (infers) is actually true - it's logically entailed by the KB.

Why Soundness Matters

Soundness ensures that our inference engine only creates true sentences:

  • ✅ No false conclusions
  • ✅ Trustworthy reasoning
  • ✅ Logical consistency
  • ❌ Prevents producing incorrect answers
🎯 Sound Inference Example

Knowledge Base: Addition rules for natural numbers

Question: $x + y = 4$ where $x, y \in \mathbb{N}$

Sound Inference produces: $x = 2, y = 2$ ✅ TRUE!

Unsound Inference might produce: $x = 1, y = 1$ ❌ FALSE!

Since $1 + 1 = 2 \neq 4$, producing $(1,1)$ as an answer would mean the inference is NOT sound.

Professor mentioned in class: "This inference i is said to be sound if whatever it produces, if sentences created by this inference i, are all true sentences. It should not give us as an answer something like $x = 1$ and $y = 1$, because that's not a sound inference. This is very important!"

4.3 Completeness

While soundness ensures we only produce true sentences, completeness ensures we can produce all true sentences.

Formal Definition

An inference mechanism i is complete if:

If KB ⊨ α, then KB ⊢i α

In words: If a sentence is logically entailed by KB, then i can produce/deduce it.

Understanding Completeness

A complete inference mechanism can find all possible true conclusions:

🎯 Complete Inference Example

Sentence: $x + y = 4$ where $x, y \in \mathbb{N}$

A COMPLETE inference engine would produce ALL solutions:

  • $x = 1, y = 3$ ✅
  • $x = 2, y = 2$ ✅
  • $x = 3, y = 1$ ✅

This is the exhaustive list of all models where the sentence is true (given natural numbers).

An INCOMPLETE inference engine might only produce:

  • $x = 2, y = 2$ ✅

This is sound (correct) but incomplete (missing other solutions).

Professor mentioned in class: "Inference i is complete if it can produce or deduce all true sentences. For $x + y = 4$, if it can create all solutions - $(1,3)$, $(2,2)$, $(3,1)$ - this is the exhaustive collection. If we are generating all this, then this i will be called complete."

Soundness vs Completeness

Property Soundness Completeness
Question Are all produced sentences true? Can all true sentences be produced?
Concern No false positives No false negatives
Guarantee Everything inferred is correct Everything correct can be inferred
Ideal System Both Sound AND Complete

The Ideal: Sound + Complete

The best inference systems are both sound and complete:

  • ✅ Sound: Never produces false conclusions
  • ✅ Complete: Can find all true conclusions
  • 🎯 Perfect reasoning system!
Professor mentioned in class: "We require the inference to be sound and complete - these are the two most important characteristics that we talked about. This is very, very important for building reliable intelligent systems."

📝 Hinglish Summary

Entailment ka matlab hai ki agar α true hai toh β bhi true hoga - yaani M(α) ⊆ M(β). Inference se hum KB se naye sentences banate hain. Do important properties hain: Soundness - jo bhi sentence produce ho wo TRUE hona chahiye (galat answer nahi), aur Completeness - sare possible TRUE sentences produce hone chahiye (koi miss nahi hona chahiye). Best inference system dono sound AUR complete hota hai!

🎯 Key Takeaways

  • Entailment (α ⊨ β) means every model of α is also a model of β
  • Inference creates new sentences from existing knowledge in the KB

  • Ideal inference systems are both sound and complete

5. Represented World vs Real World

A crucial aspect of logical agents is understanding the relationship between the real world and the represented world.

The Two Worlds

[Insert diagram: Real World ⟷ Representation ⟷ Knowledge Base ⟷ Inference ⟷ New Knowledge → Back to Real World]
Aspect Real World Represented World
Nature Physical environment, actual objects and events Symbolic representation using sentences
Components Red balls, blue balls, physical quantities Symbols like x, y, mathematical expressions
Operations Physical actions, manipulations Logical reasoning, inference
Agent Activity Perception and action Reasoning and computation

The Complete Agent Cycle

Here's how a logical agent bridges these two worlds:

  1. Real World: Complex environment with objects and events
  2. Representation: Agent creates symbolic representation using sentences
  3. Knowledge Base: Sentences stored as prior knowledge
  4. Reasoning: Inference engine derives new sentences
  5. Translation Back: New knowledge applied to real world actions
🎯 Complete Example: Red and Blue Balls

Real World Problem:

You have red balls and blue balls. The total number of balls should be 4. What combinations are possible?

Step 1: Representation (Real → Symbolic)

  • Red balls → variable $x$
  • Blue balls → variable $y$
  • Total = 4 → equation $x + y = 4$

Step 2: Knowledge Base

  • KB contains: Knowledge of addition on natural numbers
  • Constraint: $x, y \in \mathbb{N}$ (both are natural numbers)

Step 3: Reasoning (In Represented World)

Inference engine solves: $x + y = 4$

Solutions derived:

  • $x = 1, y = 3$
  • $x = 2, y = 2$
  • $x = 3, y = 1$

Step 4: Translation Back (Symbolic → Real)

  • $(1, 3)$ → 1 red ball + 3 blue balls = 4 balls total ✅
  • $(2, 2)$ → 2 red balls + 2 blue balls = 4 balls total ✅
  • $(3, 1)$ → 3 red balls + 1 blue ball = 4 balls total ✅

Step 5: Real World Action

Agent can now physically select, for example, 2 red balls and 2 blue balls to make a collection of 4 balls!

Professor mentioned in class: "In the real world, maybe you are saying we have red balls and blue balls, and when you take them together they add up to 4. How are you expressing this? In your represented world as $x + y = 4$. Then you ask your KB with knowledge of addition - what can you deduce? You get $x = 2, y = 2$. Then you translate back to real world: I need 2 red balls and 2 blue balls. That's how the agent interacts through logic!"

Why This Matters

Understanding this distinction is crucial because:

  • 🧠 Reasoning happens symbolically - Computers can't directly manipulate physical objects, they work with representations
  • 🔄 Abstraction enables generalization - The same logical rules apply to many real-world scenarios
  • ✅ Verification is possible - We can check if symbolic conclusions match real-world outcomes
  • 🎯 Efficiency - Symbolic reasoning is faster than physical trial-and-error
Professor mentioned in class: "This is where we are executing - in the real world. From this, when we ask that we need red balls plus blue balls is 4, then we are saying that this follows: you can take 2 red balls and 2 blue balls and make it 4 balls. That is how the agent is interacting through this logic."

📝 Hinglish Summary

Real world aur represented world mein farak samajhna zaroori hai. Real world mein physical objects hote hain (jaise red balls, blue balls), lekin agent apne dimag mein unhe symbols se represent karta hai ($x + y = 4$). Agent represented world mein reasoning karta hai aur phir results ko wapas real world mein translate karke action leta hai. Jaise agar inference ne bataya $(2,2)$, toh agent 2 red aur 2 blue balls lega. Yeh process logic ke through hota hai!

6. Propositional Logic

Now we dive into Propositional Logic - a simple but powerful logical system for reasoning!

Professor mentioned in class: "Now we will start with propositional logic. It is a very simple logic - simple logic. We will first talk about its syntax and semantics."

What is Propositional Logic?

Propositional logic is a formal system where:

  • Statements (propositions) can be either true or false
  • We use logical connectives to build complex statements
  • Reasoning is done through well-defined rules
  • It's the foundation for more advanced logical systems

6.1 Atomic Sentences

Atomic sentences are the simplest building blocks of propositional logic.

Definition

An atomic sentence consists of a single symbol:

Examples of Atomic Sentences
  • P - might represent "It is raining"
  • Q - might represent "I am happy"
  • R - might represent "The sun is shining"
  • x, y, p, q - any single symbol can be an atomic sentence
Professor mentioned in class: "Atomic sentences - as the name suggests, it is the simple sentence that we have. These atomic sentences consist of single symbols. Single symbol means it can be something like x, y, p, q. Instead of x, y we will have something like P, Q, or any kind of symbol that you can think of - it is a single symbol."

Special Symbols

Propositional logic has two special atomic sentences:

Symbol Name Meaning Truth Value
⊤ or TRUE Tautology Always true proposition True in every world/model
⊥ or FALSE Contradiction Always false proposition False in every world/model

Truth Values

Each atomic sentence (proposition) has two possible values:

  • ✅ True (T or 1): The proposition holds in a given model
  • ❌ False (F or 0): The proposition doesn't hold in a given model
Truth Values in Context

Sentence α: "I eat rice"

Model m₁: In a world where I am actually eating rice

→ α is TRUE in m₁

Model m₂: In a world where I am eating roti (not rice)

→ α is FALSE in m₂

Professor mentioned in class: "We can assign either T or false, true or false - these are the two values we can have. Sometimes we can denote it by numerics also - 1 and 0. If I am eating rice in that environment, it is true. If I am not eating rice, in that environment it will be false."

6.2 Logical Connectives

To form complex sentences, we use logical connectives to join atomic sentences.

The Five Main Connectives

Negation ¬ or ~ NOT ¬α "not α"
Conjunction ∧ AND α ∧ β "α and β"
Disjunction ∨ OR α ∨ β "α or β"
Implication → or ⇒ IMPLIES α → β "α implies β" or "if α then β"
Bi-implication ↔ or ⇔ IFF (if and only if) α ↔ β "α if and only if β"
Professor mentioned in class: "These are usually the 5 connectives that we use. We have negation - NOT, conjunction - AND, disjunction - OR, implication - and this is where the inference comes in, the conclusion from α that we are creating β. Then we have bi-implication - both ways, meaning that β follows from α and α also follows from β."

Literals

A literal is defined as:

  • Either an atomic sentence (positive literal), OR
  • A negated atomic sentence (negative literal)
Examples of Literals

  • ¬Q - negative literal
  • R - positive literal
  • ¬S - negative literal
Professor mentioned in class: "The literal when you are talking about a literal is either an atomic sentence - that means either a positive literal - or a negated atomic sentence. Better to call it negated atomic sentence."

6.3 Operator Precedence

Operator precedence determines the order in which operations are evaluated in complex sentences.

Precedence Order (Highest to Lowest)

Priority Operator Symbol Note
1 (Highest) Negation ¬ Evaluated first
2 Conjunction ∧ Evaluated second
3 Disjunction ∨ Evaluated third
4 Implication → Evaluated fourth
5 (Lowest) Bi-implication ↔ Evaluated last
Professor mentioned in class: "This is very, very important - the precedence of operations. It means in what order I should take operations into account. First, if there is any complex sentence and these symbols are involved, the negation (NOT) will come first, then conjunction (AND), then disjunction (OR), then implication, then bi-implication."

Why Precedence Matters

🎯 Precedence Example 1: Propositional Logic

Expression: ¬A ∧ B

❌ WRONG interpretation: ¬(A ∧ B) - "NOT (A AND B)"

✅ CORRECT interpretation: (¬A) ∧ B - "(NOT A) AND B"

Why? Negation (¬) has higher precedence than conjunction (∧), so it's evaluated first!

🎯 Precedence Example 2: Arithmetic Analogy

Expression: -2 + 4

Question: What's the answer?

Answer: 2

Process:

  1. First, evaluate negation: -2 (negative two)
  2. Then, add 4: (-2) + 4 = 2

NOT: -(2 + 4) = -6 ❌

Professor mentioned in class: "For example, when we are evaluating -2 + 4, the answer is 2, not -6. Which operation we should evaluate first and then which evaluation we should do second? Operations have some precedence. This is very important when we will be simplifying any kind of complex sentences or deriving from complex sentences another sentence."

Using Parentheses

When in doubt or when you want to override precedence, use parentheses:

  • (¬A) ∧ B - explicitly shows negation first
  • ¬(A ∧ B) - forces conjunction before negation
  • P ∨ (Q ∧ R) - forces conjunction before disjunction
  • (P ∨ Q) ∧ R - forces disjunction before conjunction

6.4 Truth Values and Semantics

Now that we understand syntax, let's explore the semantics - the meaning and truth values.

What Do Truth Values Tell Us?

Semantics decides:

  • In a particular model/world, what value should we assign to a sentence?
  • Is the sentence TRUE or FALSE in that model?
Professor mentioned in class: "Semantics decide that in a particular model or that world, what value we should give to that sentence - meaning is that: is that sentence true or is the sentence false in that particular model?"

Assigning Truth Values

Truth Value Assignment Process

Step 1: Atomic Sentences

Each atomic sentence can be assigned either T (true) or F (false) based on the model.

Example: Sentence α: "Eat rice"

  • Model m₁: Actually eating rice → α = TRUE
  • Model m₂: Eating roti instead → α = FALSE

Step 2: Complex Sentences

Use truth tables and connectives to determine complex sentence truth values.

Professor mentioned in class: "These are called the truth values that we are providing. Either it is true or it is false - these are the two values it will take. We are providing the value to that sentence - the meaning means that in that particular environment. If we are eating rice, alpha is true in that model. If we are not eating rice, alpha is false."

Q3: What is the difference between ¬(P ∧ Q) and (¬P) ∧ Q? Are they equivalent?
Answer:

They are NOT equivalent!

¬(P ∧ Q): "NOT (P AND Q)" - negation of the entire conjunction

(¬P) ∧ Q: "(NOT P) AND Q" - conjunction of negated P with Q

Example with P=T, Q=T:

  • ¬(T ∧ T) = ¬T = F
  • (¬T) ∧ T = F ∧ T = F

Example with P=T, Q=F:

  • ¬(T ∧ F) = ¬F = T
  • (¬T) ∧ F = F ∧ F = F

Different truth values → NOT equivalent!
